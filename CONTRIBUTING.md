# Contributing

## Repository Purpose

This is the public installer repository for AJ Tools. It is primarily used to:

- distribute stable installer packages
- publish tagged releases
- track public issues for released builds
- improve installation documentation and scripts

Core development, source code, and internal testing are maintained separately in the private [AJ-Tools](https://github.com/Ajmalpshaik/AJ-Tools) repository.

## What To Open Here

Open a GitHub issue in this repository when you have:

- a reproducible issue with the installer or installation process
- a problem downloading or extracting the release package
- a request for a small public-facing repository improvement
- questions about installation or uninstallation

## Before Opening An Issue

Please include the practical details needed to investigate the problem:

- Revit version
- Windows version
- AJ Tools installer version
- installation method (automated script or manual)
- a short description of the problem
- screenshots or error text if available

## Pull Requests

Pull requests are welcome for installer-facing improvements such as:

- installation script fixes
- documentation improvements
- issue template enhancements
- release automation corrections

Do not use this repository for large feature development or source code changes. Feature requests and source code contributions belong in the private [AJ-Tools](https://github.com/Ajmalpshaik/AJ-Tools) repository.

## Pull Request Expectations

Before opening a pull request:

1. Keep the scope focused on installation or distribution.
2. Update documentation if installation steps or requirements change.
3. Test installation scripts on clean test machines if possible.
4. Ensure changes align with the public installer purpose of this repository.
5. Provide clear description of the changes and their purpose.

## Release Metadata

If a change modifies public release packaging or metadata:

- keep version numbers aligned with releases
- update release notes and changelogs
- test the installer on at least one supported Revit version

## Support Boundary

If your request is actually about:

- add-in source code or features
- internal tool design or customization
- private workspace development

Please contact the maintainer directly at `ajmalnattika@gmail.com` or open an issue in the private [AJ-Tools](https://github.com/Ajmalpshaik/AJ-Tools) repository if you have access.
